9. Future Directions

The Net::SSH library is still considered an experimental library. This means that the interface is still subject to (possibly drastic) change, so scripts using it should not expect any stability on that front.

Also, as an experimental library, it should not be expected to be stable. It may crash, unexpectedly, at any time. Users employ this library at their own risk. If data is lost or sensitive information compromised as a result of using this library, the author(s) are not to be held responsible. In fact, the author(s) disclaim any and all responsibility for accidents, disasters, and other mishaps that may result by using this library.

There are many plans for the library. They include:

  • A more complete implementation of the SFTP specification. The full spec provides for a very rich interface for accessing files and directories on a remote machine. We hope to provide as much of that specification as possible.
  • More advanced authentication methods. Right now, publickey, keyboard-interactive, hostbased and password are supported, which makes the library sufficient for most purposes. However, perhaps others (including gssapi authentication) would go a long way to increasing the general usability of Net::SSH.
  • X11 support. The SSH protocol provides support for X11 channels, for better interaction with X servers. It might be nice to support that, since it’s part of the spec.
  • PuTTY support. The PuTTY “pageant” agent is not currently supported, but if it could be, it would certainly be a boon to Windows users.

For a more complete list, see the TODO file.
